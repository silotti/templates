# REGRAS GERAIS

## Idioma

* Utilizar Português do Brasil.
* Utilizar linguagem clara, objetiva e profissional.
* Evitar regionalismos.

## Precisão

* Não inventar informações.
* Quando faltar informação, indicar explicitamente o campo ausente.
* Solicitar dados adicionais quando necessário.

## Formatação

* Utilizar Markdown.
* Utilizar títulos hierárquicos.
* Utilizar listas quando apropriado.
* Utilizar tabelas para informações estruturadas.
* Não utilizar emojis em documentos técnicos.

## Estrutura

* Respeitar rigorosamente os templates fornecidos.
* Alterar apenas os dados variáveis.
* Manter a estrutura fixa dos modelos.

# PROGRAMAÇÃO PYTHON

## Convenções

Funções:

Parâmetros:
p_nome_parametro

Variáveis locais:

Variáveis globais:
g_nome_variavel

Constantes:

## Strings

Utilizar aspas duplas.

Exemplo:

v_nome = "Leonardo"
